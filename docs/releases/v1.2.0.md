# v1.2.0 — Task Harness Engine 底层架构升级

> 本次为**底层架构更新**，非功能新增。核心交付 **Task Harness Engine** —— AgentLoop 之上的跨 turn 自主循环引擎，让 Nexus 从单 turn tool loop 升级为可验证、可暂停、可恢复、可裁切上下文的自治运行时。

## 核心变更

### Task Harness Engine

Nexus runtime 之上新增的自治循环引擎，循环结构：

```
Goal → Plan → Execute (AgentLoop.runTurn) → Critique → Replan → Verify
```

每个组件均为可独立测试的纯模块：

- **TaskHarnessEngine** — 主循环装配器，负责 Goal→Plan→Execute→Critique→Replan→Verify 编排
- **GoalTracker** — 结构化任务状态（active / satisfied / blocked / max_continuations / no_progress / cancelled），通过 `thread.tags.harnessState:<runId>` 持久化，支持跨重启恢复
- **EvidenceLedger** — 证据账本，记录所有可验证的 tool/file_change/command/test/checkpoint/mcp/error 收据，支持按 `harnessRunId` 重建
- **ReadinessCritic** — 确定性就绪门禁（todo_complete / mutation_verified / no_unresolved_errors / criteria_evidence / no_storm）
- **StormBreaker** — 检测同回合重复工具调用，自动注入"change approach"指令避免死循环
- **HarnessContextManager** — 按任务相关度筛选上下文，配合 token 预算裁切，避免简单取最近 N 个
- **GoalEvaluator** — 独立模型评估目标达成度（satisfied / partial / needs_user_input / blocked），反向更新 ledger 的 supportsCriteria

### AgentLoop 扩展

- `runTurn` 签名扩展为 4 参数，支持 `RunTurnOptions`：
  - `source`: 'user' | 'harness'（harness 续跑时为 'harness'）
  - `visibleToUser`: harness 续跑时为 false，UI 不可见
  - `harnessRunId`: 关联的 harness run ID
  - `extractMemory` / `skipColdMemory` / `updateEpisode`: 控制副作用（harness 续跑时不进 cold memory）
- 新增 `persistItems` / `applyHarnessFields` 方法，自动为 harness turn 产生的 item 注入 `harnessRunId` / `harnessIteration` 字段

### Protocol 协议扩展

- 新增 `harnessContinuationItem` 类型（`threadItemSchema` discriminated union 新成员）
- 新增 `goalEvaluationSchema`（含 `criteriaEvidenceMap`）
- 给全部 15 个 item schema 加 `harnessRunId?: string` 和 `harnessIteration?: number` 可选字段

### API 层

- 新增 3 条 harness API 路由：
  - `POST /api/threads/:id/harness/start` — 预生成 harnessRunId 立即返回 202
  - `GET /api/threads/:id/harness/status` — 合并 registry 运行时状态 + GoalTracker 持久化状态
  - `POST /api/threads/:id/harness/cancel` — 通过 AbortController 取消
- 新增 `HarnessRuntimeRegistry` 进程内单例（harnessRunId → AbortController / Promise / status 映射）
- 进程退出时自动 abort 所有运行中的 harness run（`installGracefulShutdown` 新增 `onShutdown` 钩子）

### 架构整理

- 抽离 `threadRoutes.ts` 聚合器：server.ts 从 825 行降至 779 行，恢复 ≤780 架构守卫
- harness 相关 import 收敛到 threadRoutes.ts，server.ts 顶层装配更聚焦

## 修复

- 修复 `harness_continuation` item `turnId` 可能为 `undefined` 导致 storage/memory 过滤编译失败（三处加 guard）
- 修复 mid-turn 自动压缩 `soft` / `hard` 互斥逻辑错误：原逻辑要求同时满足两个条件导致 hard 永远进不去，改为只按 `hard` 触发
- 修复 `GoalTracker.load` 显式传 `runId` 时只读 `activeHarnessRunId` 而非 `harnessState:<runId>`：进程重启后可正确查询已结束 harness run 的持久化状态
