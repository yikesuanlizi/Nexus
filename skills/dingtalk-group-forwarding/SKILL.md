# 钉钉群转发 Skill

当用户要求 Nexus 或钉钉机器人把内容发送、转发、发布、通知、喊话、冒泡、艾特成员、协调群内事项，或把任务结果交付到已配置的钉钉群时，使用本 Skill。

典型触发：

- “在群里冒个泡”
- “去群里艾特一下安博魏”
- “去钉钉群说一句早上好”
- “帮我往钉钉群发：构建通过了”
- “把这段转发到群里”
- “查完部署结果后同步到群里”
- “发到群‘打完我去打DD·’一条消息：安博威的爸爸”
- 钉钉单聊里发送文件后说“把这个文件发群里”“转到群里”
- 连续对话中引用上一条钉钉群投递任务，例如“再试”“按刚才那个重发”“这条重新发”

必须使用 `dingtalk_forward_to_group` 工具完成发送。不要搜索代码、读取文件、扫描端口、调用 MCP、检查环境变量或 curl 本地 API 来寻找发送方式。

上下文引用规则：

- 连续对话中的短句引用不是缺少关键词；结合上一条明确的钉钉群投递任务恢复正文、群名和 mentions。
- 不要要求用户为了触发工具而重复说“群里/钉钉/发送”。
- 如果上下文里没有上一条明确的钉钉群投递任务，才说明无法判断要重试哪条。

工具调用建议：

```json
{
  "message": "要发送到群里的正文",
  "targetGroupName": "用户明确提到的群名，可省略",
  "mentions": ["需要真正 @ 的成员姓名，例如：付守凡；没有 @ 需求时省略"],
  "source": "dingtalk_dm",
  "intent": "send_message"
}
```

文件转发调用建议：

```json
{
  "fileMode": "current_message_files",
  "source": "dingtalk_dm",
  "intent": "send_message"
}
```

文件转发边界：

- 只用于钉钉单聊中当前或最近收到的附件转发到已配置目标群。
- 不要用于 Nexus 本地文件、路径、URL、工作区文件，或钉钉群 @ 上下文。
- 没有可用附件、下载失败、上传失败或群发失败时，工具会返回发送失败；不要假装已发送。

@ 人规则：

- 只写 `@姓名` 是普通文本，不一定触发钉钉真正 @。
- 需要 @ 人时，把成员姓名放入 `mentions`。`message` 可以包含开头的可读 `@姓名`，工具会在 Webhook Text 发送前去掉这个普通文本前缀，避免黑色假 @ 和蓝色真 @ 重复显示。
- 不要编造 staff/user ID；真实 ID 由工具通过钉钉通讯录解析。
- 成员解析顺序由工具负责：目标群已知成员缓存、钉钉通讯录搜索、组织/部门成员遍历。不要因为某人没在当前群缓存中出现就要求用户重新 @ 一次。
- 钉钉只有 Webhook Text 消息类型能稳定触发蓝色 @；Markdown、ActionCard、主动群发接口都不能作为真实 @ 的成功依据。
- 有 `mentions` 时工具必须走目标群 `sessionWebhook` 的 Text 发送路径；如果目标群还没有记录 `sessionWebhook`，让用户先在目标群 @ 机器人一次。
- 通讯录找不到成员或同名多人时，工具会返回发送失败；不要改成普通文本假装 @ 成功。

结果处理：

- 工具返回 `已发送` 时，只简短告知用户已发送。
- 工具返回 `发送失败：...` 时，只说明发送失败和原因。
- 不要向用户透露 conversationId、openConversationId、processQueryKey、messageId、access token、工具参数或内部路由细节。
